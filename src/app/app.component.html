Evaluación Desarrollador Sr.
Bienvenido a tu prueba de Desarrollador Sr. de la Federación Mexicana de Fútbol.
Estas son las actividades que debes realizar:
  1. Utilizar Angular 7 o superior.
  2. Crear una API que, más adelante, servirá para guardar y obtener información.
  3. Instalar la librería ng-wizard https://www.npmjs.com/package/ng-wizard para crear un componente por pasos.
  4. En el primer paso debes mostrar un formulario de datos generales con la siguiente información:
    • Nombre
    • Apellido Paterno
    • Apellido Materno
    • Fecha de Nacimiento (Con un campo de selección de fecha)
    • Género *
    • Nacionalidad *
    • Club de Fútbol *
    • Ocupación
    * Estas opciones deben alimentarse de manera dinámica utilizando los JSON que se encuentran en la carpeta "data", también se requiere crear los modelos de datos y guardarlos en la carpeta "models"
  5. Para el segundo paso mostrarás un nuevo formulario con datos financieros:
    • R.F.C. *
    • Calle
    • Número
    • Colonia
    • Código Postal (Validando que el número de dígitos sea de 6)
    * Este campo solo debe aparecer si el registro que se captura es de alguien mayor a 18 años, en caso que lo sea, validar la estructura del R.F.C. antes de enviar la información
  6. Utilizando la información capturada en el formulario, dentro del paso tres mostrar una ficha similar a la siguiente (Se tomará en cuenta el diseño implementado):
    Mostrar la información al usuario para verificar datos y colocar un botón que, al darle click, enviará la información a la API para su almacenamiento (puedes usar JSON)
  7. Instalar la librería https://www.npmjs.com/package/jspdf para crear PDF.
  8. En una pantalla distinta al componente de pasos, colocar un buscador por ID y por nombre
  9. Al ingresar datos, debemos consumir la API para realizar la búsqueda de información y desplegarla en un listado con la siguiente información: Nombre Completo, Fecha de Nacimiento, Género, Nacionalidad, Club de Fútbol, R.F.C. y un botón que, al dar clic sobre el, despliegue la misma información en un documento PDF.
  10. Una vez hayas concluido tu evaluación, debes subirlo a tu propio repositorio de GitHub o GitLab y enviar el link para su revisión.
